# My Project 

## 프로젝트 소개
트리(Tree) 자료구조와 허프만 코딩(Huffman Coding) 알고리즘을 활용한 파일 압축 및 압축 해제 프로젝트입니다.

입력 파일의 문자 빈도수를 계산한 뒤, 빈도수가 낮은 문자들을 먼저 결합하여 허프만 트리를 생성합니다.  
생성된 허프만 트리를 이용해 각 문자에 대한 이진 코드를 만들고, 이를 바탕으로 파일을 압축합니다.

압축 해제 시에는 압축 파일에 저장된 빈도수 정보를 이용하여 다시 허프만 트리를 복원하고, 압축된 비트를 읽어 원본 파일로 복원합니다.

## 동작 과정

### 1단계: 빈도수 계산

입력 파일을 한 글자씩 읽으면서 각 문자가 몇 번 등장했는지 계산합니다.
계산된 빈도수는 `freq` 배열에 저장됩니다.

### 2단계: 초기 노드 생성

빈도수가 0보다 큰 문자들만 노드로 생성합니다.
각 노드는 문자, 빈도수, 왼쪽 자식, 오른쪽 자식 정보를 가집니다.

### 3단계: 허프만 트리 생성

빈도수가 가장 작은 노드 2개를 선택하여 새로운 부모 노드를 만듭니다.
이 과정을 하나의 루트 노드가 남을 때까지 반복합니다.

### 4단계: 부호표 생성

완성된 허프만 트리를 탐색하면서 각 문자에 대한 이진 코드를 생성합니다.

* 왼쪽으로 이동하면 0
* 오른쪽으로 이동하면 1

이 규칙을 이용해 각 문자마다 고유한 코드가 만들어집니다.

### 5단계: 파일 압축

원본 파일을 다시 읽으면서 각 문자를 부호표에 있는 이진 코드로 변환합니다.
변환된 비트들은 8비트씩 묶여 압축 파일에 저장됩니다.

압축 파일에는 압축 해제를 위해 문자 빈도수 정보도 함께 저장됩니다.

### 6단계: 압축 해제

압축 파일에 저장된 빈도수 정보를 읽어 허프만 트리를 다시 생성합니다.
이후 압축된 비트를 하나씩 읽으며 트리를 따라가고, 리프 노드에 도착하면 해당 문자를 복원합니다.

## 어려웠던 점

허프만 트리를 만들 때 가장 작은 빈도수의 노드 2개를 선택하는 과정이 어려웠다.
압축할 때 문자 코드를 8비트씩 묶어서 파일에 저장하는 부분이 복잡했다.
압축 해제 과정에서 패딩 비트를 제외하고 정확히 원본 파일로 복원하는 부분이 어려웠다.
포인터를 이용해 트리의 왼쪽 자식과 오른쪽 자식을 연결하는 과정에서 메모리 관리에 주의해야 했다.

## 느낀 점

이번 프로젝트를 통해 트리 자료구조가 단순히 이론적인 구조가 아니라 실제 파일 압축 알고리즘에 활용될 수 있다는 것을 알게 되었다.
특히 허프만 코딩은 문자의 등장 빈도에 따라 자주 나오는 문자는 짧은 코드로, 적게 나오는 문자는 긴 코드로 표현하여 파일 크기를 줄이는 방식이라는 점이 인상 깊었다.
또한 트리뿐만 아니라 큐, 스택, 파일 입출력, 포인터, 구조체 등 C언어에서 배운 여러 개념을 함께 활용할 수 있어서 자료구조의 실제 사용 방법을 이해하는 데 도움이 되었다.
